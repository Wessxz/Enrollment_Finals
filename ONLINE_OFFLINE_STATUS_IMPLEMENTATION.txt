================================================================================
                ONLINE/OFFLINE STATUS IMPLEMENTATION GUIDE
================================================================================

OVERVIEW:
This system prevents multiple simultaneous logins for the same user account.
When a user logs in, they are marked as "online" and cannot login again until
they logout or are manually set to offline by an admin.

================================================================================
STEP 1: DATABASE CHANGES
================================================================================

Run these SQL commands in your database:

-- Add is_online column (0 = offline, 1 = online)
ALTER TABLE users 
ADD COLUMN is_online TINYINT(1) DEFAULT 0 AFTER role;

-- Add last_login tracking
ALTER TABLE users 
ADD COLUMN last_login DATETIME NULL AFTER is_online;

-- Set all existing users to offline
UPDATE users SET is_online = 0;

-- Optional: Drop old status column if not needed for staff
-- (Keep it if you still need it, or drop it after testing)
ALTER TABLE users DROP COLUMN status;


================================================================================
STEP 2: HOW IT WORKS
================================================================================

LOGIN FLOW:
-----------
1. User enters username and password
2. System checks if credentials are correct
3. System checks if user is already online (is_online = 1)
   - If YES: Show error "Already logged in on another device"
   - If NO: Continue to step 4
4. For students: Check if verified in students table
5. Set user to online (is_online = 1)
6. Record login time (last_login = NOW())
7. Open appropriate dashboard

LOGOUT FLOW:
------------
1. User clicks Logout button
2. System sets user to offline (is_online = 0)
3. Return to login screen

CLOSE APP WITHOUT LOGOUT:
--------------------------
If user closes the app without clicking logout, they will remain "online"
in the database. To handle this:
- Admin can manually force logout
- Or implement auto-logout after timeout (future enhancement)

================================================================================
STEP 5: USER TABLE STRUCTURE
================================================================================

FINAL users TABLE STRUCTURE:
----------------------------
- id (INT, PRIMARY KEY)
- username (VARCHAR)
- password_hash (VARCHAR)
- role (ENUM: admin, registrar, cashier, faculty, student)
- is_online (TINYINT: 0 or 1) ← NEW
- last_login (DATETIME) ← NEW
- created_at (DATETIME)

FINAL students TABLE STRUCTURE:
-------------------------------
- id (INT, PRIMARY KEY)
- student_id (VARCHAR, e.g., 2026-0001)
- first_name (VARCHAR)
- last_name (VARCHAR)
- course (VARCHAR)
- year_level (VARCHAR)
- student_type (ENUM: REGULAR, IRREGULAR)
- section_id (INT, nullable)
- status (ENUM: pending, verified, rejected) ← For verification
- user_id (INT, FOREIGN KEY to users.id)


================================================================================
STEP 6: VERIFICATION LOGIC
================================================================================

STAFF ACCOUNTS (admin, registrar, cashier, faculty):
----------------------------------------------------
- Created by admin through desktop app
- No verification needed
- Can login immediately
- Only checked: username, password, is_online

STUDENT ACCOUNTS:
-----------------
- Created via web portal registration
- Needs admin verification
- Cannot login until verified
- Checked: username, password, students.status='verified', is_online

================================================================================
STEP 8: TROUBLESHOOTING
================================================================================

PROBLEM: User can't login, says "already logged in"
SOLUTION: Admin manually sets is_online = 0 for that user

PROBLEM: All users stuck as online
SOLUTION: Run this SQL command:
UPDATE users SET is_online = 0;

PROBLEM: Student can't login after verification
SOLUTION: Check students.status = 'verified' and users.is_online = 0

PROBLEM: Staff can't login
SOLUTION: Check users.is_online = 0 (staff don't need verification)


================================================================================
STEP 9: SQL QUERIES FOR ADMIN
================================================================================

-- View all online users
SELECT id, username, role, last_login 
FROM users 
WHERE is_online = 1;

-- Force logout a specific user
UPDATE users SET is_online = 0 WHERE username = 'cashier';

-- Force logout all users
UPDATE users SET is_online = 0;

-- View students pending verification
SELECT s.student_id, s.first_name, s.last_name, s.status
FROM students s
WHERE s.status = 'pending';

-- Verify a student
UPDATE students SET status = 'verified' WHERE student_id = '2026-0001';
